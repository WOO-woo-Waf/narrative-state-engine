# 对话主入口迁移与测试计划

## 一、迁移原则

本次重构不要一次性推倒全部功能。

迁移方式：

```text
保留现有 API 和状态机
新增对话主入口
把现有分析、审计、状态、图谱接入对话线程
逐步降低旧多面板页面权重
```

关键原则：

```text
状态机优先。
旧代码保留。
图谱保留并增强。
对话入口是新的操作层，不是新的状态来源。
```

禁止：

```text
删除旧候选审计能力。
删除旧状态环境接口。
删除旧图谱接口。
新增一套与状态机平行的小说状态存储。
让对话线程保存不可追踪的状态副本。
```

## 二、阶段拆分

### 阶段一：对话主界面骨架

前端：

```text
左侧小说/任务
中间对话线程
底部输入框
右侧上下文摘要
```

后端：

```text
对话线程
消息存储
上下文环境接口
非执行型助手回复
```

验收：

```text
能选择小说和任务。
能发送消息。
能保存线程。
能看到当前上下文摘要。
```

同时验收：

```text
旧 workbench 入口仍可打开。
旧状态环境接口仍正常。
旧候选列表接口仍正常。
旧图谱接口仍正常。
```

新增重点：

```text
对话压缩只压缩消息历史，不改变状态机。
场景切换只切换上下文视图，不复制状态。
对话线程可以引用状态对象、候选、迁移和分支。
```

### 阶段二：分析任务接入

前端：

```text
分析任务草案卡片
执行按钮
运行卡片
分析结果 artifact
```

后端：

```text
分析任务草案
分析任务执行
source role 策略
结果写入候选集合
```

验收：

```text
作者能通过对话创建分析任务。
确认后能执行。
执行结果回到对话线程。
候选集合可见。
```

同时验收：

```text
分析任务通过动作草案确认后执行。
分析结果进入现有 source/evidence/candidate_set。
对话线程只保存分析 artifact 和运行事件。
```

### 阶段三：审计任务接入

前端：

```text
审计上下文摘要
模型审计输入
审计草案卡片
执行结果卡片
候选辅助页
```

后端：

```text
候选风险评估
审计草案生成
草案执行
状态迁移记录
图谱刷新标记
```

验收：

```text
作者能让模型生成多份审计草案。
作者能执行一份草案。
状态对象和迁移更新。
结果回到对话线程。
```

同时验收：

```text
审计执行产生的状态对象能在状态对象页看到。
审计执行产生的迁移能在迁移图看到。
对话结果卡片可以跳转到对应候选和迁移。
```

同时验收：

```text
模型生成的审计建议先保存为草案。
作者确认后才批量执行。
执行结果进入现有 state_objects/state_transitions。
失败、跳过、冲突都必须在对话线程中展示。
```

### 阶段四：规划和续写接入

前端：

```text
规划草案卡片
续写任务草案卡片
草稿分支 artifact
分支审稿入口
```

后端：

```text
剧情规划上下文
续写上下文
生成任务
分支存储
分支审稿
```

验收：

```text
作者能通过对话规划下一章。
确认后能创建续写任务。
生成分支能回到对话线程。
```

## 三、真实数据测试

使用：

```text
小说编号：story_123_series_realrun_20260510
任务编号：task_123_series_realrun_20260510
```

第一轮只测试：

```text
打开工作台
选择小说/任务
查看上下文摘要
发送审计消息
生成审计草案
执行低风险草案
查看执行结果
查看状态迁移
```

## 四、通过标准

最低通过：

```text
主界面是对话线程。
输入框是主入口。
候选表不是默认主界面。
分析和审计都能通过对话生成草案。
作者确认后才能执行。
执行结果能回到对话线程。
中文界面可读。
```

状态机相关通过标准：

```text
所有分析、审计、规划、续写结果最终都引用或回写统一状态环境。
对话线程不形成第二套小说设定。
图谱能展示状态发展、关系、迁移和分支。
旧代码路径可作为回退方案继续使用。
```

推荐通过：

```text
运行过程像 Codex 一样逐步展示。
草案可以修改。
失败项可以回看。
状态图和迁移图可以从结果卡片打开。
上下文摘要足够清楚，不需要作者翻 85 条候选。
```
