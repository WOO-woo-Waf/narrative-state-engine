# 前端执行计划：剧情规划、续写、分支审稿与状态回流

本文档给前端窗口使用，目标是在已完成的对话主入口基础上，把后半条真实写作链路做成一等场景：

```text
剧情规划
  -> 续写任务
  -> 生成分支
  -> 分支审稿
  -> 修改/重写/接受/丢弃
  -> 状态回流审计
```

前端不要再把这些能力隐藏成普通工具按钮。它们要像“候选审计”一样成为对话工作台中的明确场景。

## 一、当前基础

已完成能力：

```text
/workbench-v2/workbench-dialogue/
对话主入口
线程列表
动作草案卡片
确认执行
运行卡片
artifact 卡片
候选/状态/图谱/证据/分支辅助工作区
图谱高亮
旧工作台回退
```

后端已提供相关工具：

```text
create_plot_plan
create_generation_job
review_branch
accept_branch
reject_branch
rewrite_branch
```

本次前端重点：

```text
把这些工具包装成作者能理解的剧情规划、续写、分支审稿场景。
```

## 二、场景导航

左侧场景列表必须新增或明确展示：

```text
候选审计
剧情规划
续写任务
分支审稿
状态回流
```

点击场景后：

```text
对话线程仍在中间。
底部输入框提示语变化。
右侧上下文摘要变化。
可用快捷指令变化。
可用工具草案变化。
```

## 三、剧情规划场景

### 三点一、入口

场景名称：

```text
剧情规划
```

scene_type 建议：

```text
plot_planning
```

### 三点二、上下文摘要

右侧上下文摘要显示：

```text
当前状态版本
已确认角色
关键世界规则
未解决剧情线
未回收伏笔
作者锁定字段
最近审计结果
可用参考证据
```

不要展示完整 JSON，默认只展示摘要，支持“展开详情”。

### 三点三、输入框提示

占位文本：

```text
告诉模型你想如何规划后续剧情。例如：生成三种下一章走向，不写正文，只规划场景和状态变化。
```

快捷指令：

```text
生成三种下一章规划
保守推进主线
强化冲突
整理伏笔
合并两个规划
列出规划风险
```

### 三点四、规划草案卡片

模型返回 `create_plot_plan` 草案时，前端渲染为：

```text
剧情规划草案
```

卡片字段：

```text
规划标题
规划摘要
章节目标
场景顺序
出场角色
角色状态变化
世界规则使用
伏笔推进
禁止破坏项
风险等级
需要作者确认的问题
```

按钮：

```text
查看详情
执行规划
让模型修改
合并为新规划
取消草案
```

确认文本按后端返回：

```text
确认执行
```

### 三点五、规划 Artifact

执行成功后显示：

```text
剧情规划结果
```

Artifact 操作：

```text
查看详情
用于续写
打开相关状态
打开图谱
```

如果 artifact 有 `plot_plan_id`，前端需要保存当前选中的规划，续写场景默认引用它。

## 四、续写任务场景

### 四点一、入口

场景名称：

```text
续写任务
```

scene_type 建议：

```text
continuation
```

### 四点二、上下文摘要

右侧显示：

```text
当前状态版本
当前选中剧情规划
角色状态摘要
世界规则摘要
伏笔摘要
风格样例摘要
参考证据范围
上下文预算
禁止破坏项
```

如果没有已确认规划，显示：

```text
当前没有已确认剧情规划。建议先进入“剧情规划”场景生成规划。
```

### 四点三、续写上下文预览

新增卡片：

```text
续写上下文预览
```

按钮：

```text
预览续写上下文
刷新上下文
展开证据
展开角色
展开世界规则
```

如果后端提供 `preview_generation_context`，前端调用该工具或对应接口。

如果后端暂未提供，前端先用现有 environment/context 数据显示降级预览，并标明：

```text
当前为降级预览，未调用完整续写上下文工具。
```

### 四点四、续写草案卡片

模型返回 `create_generation_job` 草案时，渲染为：

```text
续写任务草案
```

字段：

```text
使用规划
使用状态版本
生成模式
分支数量
字数范围
上下文预算
RAG 策略
参考文本策略
是否生成后状态回流
风险提示
```

按钮：

```text
查看详情
执行生成
修改参数
取消草案
```

执行前确认：

```text
确认执行
```

### 四点五、运行卡片

续写执行过程要在对话线程里展示：

```text
正在构建续写上下文
正在读取主状态
正在读取作者规划
正在检索参考证据
正在生成章节蓝图
正在生成草稿分支
正在保存分支 artifact
等待作者审稿
```

每一步可以来自后端 events。如果后端只返回最终 artifact，前端也要用已有事件和状态做最小运行卡片。

### 四点六、分支 Artifact

生成成功后显示：

```text
续写分支
```

字段：

```text
分支编号
生成时间
基础状态版本
使用规划
正文预览
输出路径
风险提示
```

按钮：

```text
打开分支
审稿分支
重写分支
丢弃分支
```

点击“审稿分支”应切换到“分支审稿”场景，并带上 `branch_id`。

## 五、分支审稿场景

### 五点一、入口

场景名称：

```text
分支审稿
```

scene_type 建议：

```text
branch_review
```

### 五点二、分支选择

右侧或顶部显示：

```text
当前分支
分支列表
基础版本
主线版本
是否存在版本漂移
```

无分支时显示：

```text
当前没有续写分支。请先进入“续写任务”生成分支。
```

### 五点三、审稿输入提示

占位文本：

```text
告诉模型如何审稿。例如：检查这个分支是否符合当前状态、规划和风格，列出风险，不要入库。
```

快捷指令：

```text
检查状态一致性
检查风格一致性
列出需要重写的问题
生成重写草案
接受分支入主线
丢弃分支
```

### 五点四、审稿报告 Artifact

`review_branch` 执行后渲染：

```text
分支审稿报告
```

字段：

```text
一致性评分
风格匹配
状态破坏风险
剧情规划符合度
需要重写的段落
建议动作
```

按钮：

```text
查看分支
重写分支
接受入库
丢弃分支
打开分支图
```

### 五点五、分支接受确认

`accept_branch` 必须使用后端确认文本：

```text
确认入库
```

前端执行前显示高风险提示：

```text
接受分支会影响主线正文和后续状态。执行后仍需要进行状态回流审计。
```

## 六、状态回流场景

### 六点一、触发时机

分支接受后，前端必须显示：

```text
正文已接受。是否分析本章新增状态并生成状态回流审计草案？
```

按钮：

```text
生成状态回流审计
稍后处理
查看已接受分支
```

### 六点二、状态回流提示

如果后端尚未提供完整状态回流工具，前端仍要展示占位流程：

```text
状态回流功能待后端完成。
当前已记录分支接受结果，可稍后从状态回流场景继续。
```

如果后端提供工具，则调用：

```text
analyze_generated_branch_for_state_updates
create_branch_state_review_draft
```

### 六点三、回流审计结果

生成后显示：

```text
状态回流审计草案
```

并复用候选审计工作区：

```text
查看候选
执行低风险回流
保留高风险回流
打开迁移图
```

## 七、辅助工作区要求

### 七点一、图谱

从规划、续写、分支审稿 artifact 打开图谱时，前端应自动选择合适图：

| 来源 | 默认图谱 |
| --- | --- |
| 规划 artifact | 状态图 / 关系图 |
| 续写分支 artifact | 分支图 |
| 审稿报告 artifact | 分支图 |
| 状态回流 artifact | 迁移图 |

高亮：

```text
plot_plan_id
branch_id
transition_id
state_object_id
candidate_item_id
```

### 七点二、分支工作区

分支工作区必须能从 artifact 打开。

显示：

```text
分支正文预览
分支状态
基础版本
主线版本
审稿报告
操作按钮
```

### 七点三、候选工作区

状态回流时复用已有候选工作区，但标题要明确：

```text
状态回流候选审计
```

不要和原始分析候选混淆。

## 八、E2E 测试要求

新增 e2e：

```text
dialogue planning continuation flow
```

覆盖：

1. 打开 `/workbench-v2/workbench-dialogue/`。
2. 选择 story/task。
3. 切换“剧情规划”。
4. 输入规划请求。
5. 渲染 `create_plot_plan` 草案。
6. 确认执行规划草案。
7. 显示剧情规划 artifact。
8. 点击“用于续写”。
9. 切换“续写任务”。
10. 显示续写上下文预览。
11. 渲染 `create_generation_job` 草案。
12. 执行生成。
13. 显示续写分支 artifact。
14. 点击“审稿分支”。
15. 切换“分支审稿”。
16. 渲染 `review_branch` 结果。
17. 点击“接受入库”时要求确认文本“确认入库”。
18. 显示状态回流提示。
19. 打开分支图并高亮 branch_id。

## 九、真实测试建议

真实 123 测试入口：

```text
/workbench-v2/workbench-dialogue/
```

使用：

```text
story_123_series_realrun_20260510
task_123_series_realrun_20260510
```

建议第一轮只生成一个短分支：

```text
分支数量：1
字数范围：较小
先重视链路，不重视最终文本质量
```

测试重点：

```text
规划是否能保存
续写是否引用规划
上下文预览是否清楚
分支 artifact 是否出现
审稿报告是否出现
接受分支是否要求确认入库
状态回流提示是否出现
图谱是否刷新并高亮
```

## 十、完成标准

最低完成：

```text
剧情规划、续写任务、分支审稿、状态回流四个场景可见。
每个场景有中文输入提示和快捷指令。
规划草案、续写草案、审稿报告、分支 artifact 能在对话线程中展示。
续写前有上下文预览。
分支接受要求“确认入库”。
分支接受后有状态回流提示。
E2E 覆盖后半链路。
```

推荐完成：

```text
支持规划草案合并。
支持续写参数编辑。
支持分支重写草案。
支持状态回流候选审计。
支持从每个 artifact 打开并高亮对应图谱。
```
